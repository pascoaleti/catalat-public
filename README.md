Catalat is a catalog website SaaS for businesses that need a fast public storefront, custom domain support and a private admin panel without the complexity of a full ecommerce checkout.

The product is focused on contact-driven sales: products, cars, real estate and local businesses where the buyer usually wants to talk, ask questions, negotiate or close through WhatsApp, phone, email or a direct channel.

## Why this public repository exists

This repository is the public institutional profile for Catalat.

It exists to document the product, provide official links and make the project easier to evaluate from GitHub without exposing the production SaaS source code.

The private production repository remains closed because it contains the real application code, operational logic, deployment details and integrations.

## Product highlights

- Public catalog websites with clean navigation and SEO-friendly pages.
- Private admin panel for products, categories, pages, blog posts and site settings.
- Custom domain flow with DNS guidance and SSL automation.
- Multilingual public website and customer panel.
- Contact-first conversion flow for WhatsApp, email and direct negotiation.
- Performance-oriented public pages for customer catalogs.

## Repository policy

This is not the open-source codebase of the SaaS.

Issues and discussions can be used for product feedback, security contact routing and public questions. Pull requests that attempt to add product source code are not expected here.

## Security

If you believe you found a security issue, do not open a public issue with sensitive details.

Use the contact flow described in [SECURITY.md](SECURITY.md).

The public website and customer-facing panel support multiple languages. This repository keeps the project profile in English because it is intended for wider GitHub visibility.
